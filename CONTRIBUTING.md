# Contributing to OpenForge

Thanks for your interest in contributing. OpenForge is a transparent, AI-assisted lab. Please keep changes clean, documented, and consistent.

## Ground Rules
- Keep changes small and focused.
- Match the existing design system and naming patterns.
- Document AI assistance in `AI_CONTRIBUTION.md` and `.ai_logs/`.
- Update the manifest when a public artifact changes.

## How to Contribute
1. Fork the repository.
2. Create a feature branch.
3. Make changes and update docs.
4. Open a pull request with a short summary.

## Documentation Requirements
- Update `README.md` when behavior changes.
- Update `docs/` for architecture or security changes.
- Add screenshots or icons for AppStore-visible changes.

## Reporting Issues
- Use clear titles and steps to reproduce.
- Include environment details (OS, runtime, version).

## Code Style
- Prefer explicit names over clever abbreviations.
- Avoid mixing tabs and spaces.
- Keep Markdown readable and skimmable.
