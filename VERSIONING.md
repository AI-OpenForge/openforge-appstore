# Versioning Policy

OpenForge projects should use semantic versioning where possible.

## Format
`MAJOR.MINOR.PATCH`

## Guidelines
- MAJOR: breaking changes
- MINOR: backward-compatible features
- PATCH: backward-compatible fixes

## Alternatives
If a project is experimental, you may use date-based versions:
`YYYY.MM.DD`

## Manifest
Always keep `catalog.json` in sync with the latest published version.
